# Amplitude → Snowflake Destination — Setup Reference

A working record of how I connected Amplitude's Snowflake destination to my Snowflake
account, including the exact SQL used and the gotchas I hit.

Completed: 2026-07-17

---

## Overview

Amplitude pushes event data into *my* Snowflake account on a recurring micro-batch
(~every 10 minutes). To receive it, Snowflake needs four dedicated objects — a
**warehouse**, **database**, **role**, and **user** — plus grants tying them together,
and a **key pair** for authentication.

The work happens in two places in parallel:
- **Snowflake** (Snowsight → Worksheets): create the objects, register Amplitude's
  public key, run the grants.
- **Amplitude** (Data → Catalog → Destinations → Snowflake): fill in connection fields,
  generate the key pair, run Test Access.

Key realization: **Amplitude generated the key pair itself** (via the **Generate Key**
button). Amplitude keeps the private key; it only shows the *public* key inside its
autogenerated SQL. So there was **no need to generate a key in my terminal** — the local
`openssl` keys I made early on were never used and can be deleted.

| Field | Value |
|---|---|
| Account Identifier | `ZTZMJYI-GA57724` (format `ORGNAME-ACCOUNTNAME`, no region suffix) |
| Warehouse | `AMPLITUDE_WH` |
| Database | `AMPLITUDE_DB` |
| Role | `AMPLITUDE` |
| User | `AMPLITUDE_USER` |

Confirm the account identifier any time with:

```sql
SELECT CURRENT_ORGANIZATION_NAME() || '-' || CURRENT_ACCOUNT_NAME() AS amplitude_account_identifier;
```

## Steps completed

### 1. Create the Snowflake objects (Snowsight worksheet, role = ACCOUNTADMIN)

```sql
-- Warehouse (dedicated to Amplitude; small + aggressive auto-suspend to save credits)
CREATE WAREHOUSE IF NOT EXISTS AMPLITUDE_WH
  WITH
    WAREHOUSE_SIZE = 'XSMALL'
    AUTO_SUSPEND   = 60
    AUTO_RESUME    = TRUE
    INITIALLY_SUSPENDED = TRUE
    COMMENT = 'Dedicated warehouse for Amplitude event exports';

-- Database (dedicated to Amplitude)
CREATE DATABASE IF NOT EXISTS AMPLITUDE_DB;

-- Role (Amplitude UI default name is AMPLITUDE)
CREATE ROLE IF NOT EXISTS AMPLITUDE;

-- Service user
CREATE USER IF NOT EXISTS AMPLITUDE_USER
  DEFAULT_ROLE      = AMPLITUDE
  DEFAULT_WAREHOUSE = AMPLITUDE_WH
  COMMENT = 'Service account for Amplitude Snowflake export';

GRANT ROLE AMPLITUDE TO USER AMPLITUDE_USER;
```

### 2. Fill in the Amplitude fields + Generate Key

In Amplitude (Data → Catalog → Destinations → Snowflake), enter the values from the table
above, choose **Key pair**, and click **Generate Key**. Amplitude then shows an
autogenerated SQL snippet on the right that contains its **public key**.

### 3. Register Amplitude's public key on the user

Amplitude's autogenerated snippet can't be run as-is (it re-`CREATE`s objects that already
exist, and it wraps string values in double quotes, which Snowflake rejects). Since the
objects already existed, I only needed to point the user at Amplitude's public key — using
**single quotes** and `ALTER` instead of `CREATE`:

```sql
ALTER USER "AMPLITUDE_USER" SET
  RSA_PUBLIC_KEY='<PASTE AMPLITUDE PUBLIC KEY BODY HERE>'
  TYPE = SERVICE;
```

> The public key value comes straight from Amplitude's autogenerated SQL panel
> (the `RSA_PUBLIC_KEY = "..."` line). Paste only the base64 body, no
> `BEGIN/END` lines, wrapped in single quotes.

### 4. Grant the role access (role = ACCOUNTADMIN)

```sql
GRANT USAGE         ON WAREHOUSE "AMPLITUDE_WH" TO ROLE "AMPLITUDE";
GRANT USAGE         ON DATABASE  "AMPLITUDE_DB" TO ROLE "AMPLITUDE";
GRANT CREATE SCHEMA ON DATABASE  "AMPLITUDE_DB" TO ROLE "AMPLITUDE";
GRANT ROLE "AMPLITUDE" TO USER "AMPLITUDE_USER";
```

### 5. Verify, then run Test Access

```sql
-- Confirm the key + type on the user
DESC USER AMPLITUDE_USER;
--   RSA_PUBLIC_KEY should match Amplitude's current key
--   RSA_PUBLIC_KEY_FP should be freshly set
--   TYPE = SERVICE, HAS_KEYPAIR = true

-- Confirm objects exist
SHOW WAREHOUSES LIKE 'AMPLITUDE_WH';
SHOW DATABASES  LIKE 'AMPLITUDE_DB';

-- Confirm grants on the role
SHOW GRANTS TO ROLE AMPLITUDE;
--   USAGE on AMPLITUDE_WH, USAGE + CREATE SCHEMA on AMPLITUDE_DB
```

Back in Amplitude, click **Next** to run **Test Access**. On success, click **Finish**.

## Gotchas / lessons learned

1. **`JWT token is invalid` = key mismatch.** Snowflake was trusting a different public key
   than the private key Amplitude was signing with. Fix: re-run the `ALTER USER ... SET
   RSA_PUBLIC_KEY='...'` with Amplitude's *current* key.
2. **Every click of "Generate Key" mints a new pair** and invalidates whatever is in
   Snowflake. After generating, register that key and finish the test in one sitting —
   don't click Generate Key again.
3. **Snowsight runs only the statement under the cursor.** Pasting a block and hitting Run
   executes just one statement. Use **Run All** (dropdown by the Run button) or select all
   first — otherwise objects/grants silently don't get created.
4. **`Object does not exist, or operation cannot be performed`** is ambiguous — it means
   either the object truly doesn't exist *or* the role lacks privileges to see it. Check
   both with `SHOW DATABASES` / `SHOW GRANTS`.
5. **Single vs double quotes:** Snowflake uses single quotes for string literals
   (`'SMALL'`, `'...key...'`) and double quotes for identifiers (`"AMPLITUDE_DB"`).
   Amplitude's autogenerated SQL double-quotes string values, which errors — swap to single
   quotes if you ever run it.
6. **No terminal key needed.** Amplitude's Generate Key produces the pair; the local
   `openssl`-generated keys were unused. Safe to remove:
   `rm ~/amplitude_rsa_key.p8 ~/amplitude_rsa_key.pub`.

## What happens next

- Amplitude sends all future events automatically, batched ~every 10 minutes; data
  typically appears within ~20 minutes.
- Data lands in objects auto-named from the Amplitude org/project IDs (not `AMPLITUDE_DB`
  directly):
  - Events: `DB_{org_id}.SCHEMA_{project_id}.EVENTS_{project_id}`
  - Merged users: `MERGE_IDS_{project_id}`
- To pull in existing data, use **Export Data** in the destination and choose a date range
  for a historical backfill.
- The warehouse auto-suspends after 60s idle, so it won't quietly burn credits between
  batches.
